Kendo Grid有相當多設定，我們會先從Datasource這一個部份開始看起

1. data - 資料來源

這邊是存放Grid需呈現資料的地方，或是未來也可以提供API路由(參考連結設定)

2. pageSize

設定一頁要顯示的筆數

3. schema - 定義資料格式

這邊可以針對資料類型(bool、string、date)進行定義，讓套件可以自動幫你做對應類型的處理，也可以設定資料驗證

接下來我們進行Columns定義

1. field

定義要顯示Datasource中的哪一個欄位

2. title

3. format

如果有數字、日期格式，可以定義呈現的格式

4. command

需要有編輯功能，需要用此屬性增加更新、刪除鈕

常用的事件

在對應的屬性加上寫好的 function callback，即會在對應的時間點觸發
- dataBinding - 在資料繫結之前觸發
- dataBound - 在資料繫結時觸發
- change - 在選取 grid 中的某列、欄時觸發
- save - 單筆存檔時觸發
- saveChanges - 批次存檔時觸發
- edit - 新增、編輯資料時觸發
更多請看這裡

常用的方法

透過 selector 取得 grid 元素後調用欲使用的方法
- addRow - 插入一筆資料 (限 inline、incell 模式)
- editRow - 讓指定的 row 轉成 edit 模式 (限 inline、incell 模式)
- items - 取得目前 grid 上的所有資料樣本
- refresh - 對當前的 grid 中的資料(items)進行重新整理
- cancelChanges - 通常在批次處理(Batch)的模式下調用，可以 rollback 所有變更
- select - 取得或設置被選取的 rows、cells
